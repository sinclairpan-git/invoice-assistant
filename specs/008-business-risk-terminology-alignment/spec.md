**创建日期**：2026-04-19
**状态**：已完成实现与归档收口（2026-04-19）
**输入**：按对抗式合议裁决，统一用户可见术语到“业务风险分类”：以 specs/001 与 2026-04-17 设计基线为 authoritative source，只收口 UI 文案、Excel manifest 列头和相关规格/测试，不改内部或 API 键 business_compliance_status，不改数据库 schema、规则逻辑。 参考：`docs/invoice-assistant-gap-backlog.zh-CN.md`、`specs/001-invoice-assistant-mvp/spec.md`、`specs/004-controlled-review-export/spec.md`、`docs/superpowers/specs/2026-04-17-invoice-assistant-design.md`

- 覆盖受影响测试断言与当前正式规格文案中的术语漂移收口。
- 覆盖本 work item 的 formal docs、任务分解与执行归档。

- 不覆盖数据库 schema、规则判定逻辑、API 响应键或历史执行日志改写。

## 用户场景与测试（必填）

### 用户故事 1 - 界面与导出要使用权威术语（优先级：P0）

作为复核与导出责任人，我希望界面和 Excel 台账统一使用“业务风险分类”这一权威术语，这样我不会把“业务合规”和“业务风险分类”误认为两套不同概念。

**优先级说明**：`001` 和设计基线都已明确裁决“必须使用业务风险分类，而不是业务合规”；当前 UI 与导出产物的口径漂移会直接误导用户。

**独立测试**：前端测试与导出服务测试分别断言详情文案和 Excel 列头已统一成“业务风险分类”。

**验收场景**：

1. **Given** 用户打开发票详情抽屉，**When** 查看解释区块，**Then** 标签显示“业务风险分类”，不再显示“业务合规”。
2. **Given** 用户导出 `excel_manifest`，**When** 查看台账列头，**Then** 列头显示“业务风险分类”，不再显示“业务合规”。

---

### 用户故事 2 - 正式规格不能继续互相冲突（优先级：P1）

作为后续维护者，我希望当前生效的规格文案与上位基线保持一致，这样下一轮实现不会再被旧 wording 拉回“业务合规”。

**优先级说明**：这不是运行时 blocker，但它是反复漂移的根源。若不修正文档，后续 UI/导出术语很容易再次回退。

**独立测试**：规格文档对账 + `python -m ai_sdlc workitem close-check --wi ... --json`

**验收场景**：

1. **Given** 维护者查看 `001`、`002`、`004` 当前正式规格， **When** 比较用户可见术语， **Then** 不再出现互相冲突的“业务合规/业务风险分类”表述。
2. **Given** 当前 work item 已归档， **When** 后续执行 `close-check`， **Then** 文档一致性不再把旧术语漂移带回实现层。

---

### 边界情况

- 内部/API 键 `business_compliance_status` 本轮保留，不作为术语统一对象。
- 历史执行日志中的“业务合规”属于历史事实，本轮不重写。
- 术语统一仅限用户可见文案和当前正式规格，不扩展到规则逻辑重命名。

## 需求（必填）

### 功能需求

- **FR-001**：用户可见术语必须统一使用“业务风险分类”，不得继续把同一概念显示为“业务合规”。
- **FR-002**：发票详情抽屉中当前显示“业务合规”的标签必须改为“业务风险分类”。
- **FR-003**：`excel_manifest` 列头中当前显示“业务合规”的列必须改为“业务风险分类”。
- **FR-004**：受影响测试必须锁定新术语，避免后续再次漂回“业务合规”。
- **FR-005**：当前正式规格中与用户可见术语相关的漂移表述必须与 `001` 和设计基线对齐。
- **FR-006**：系统不得在本轮中修改 `business_compliance_status` 内部字段名、API 键或数据库 schema。

### 关键实体（如涉及数据则必填）

- **Authoritative Terminology**：由 `specs/001` 与 2026-04-17 设计基线裁定的用户可见术语“业务风险分类”。
- **User-visible Surface**：本轮术语统一的实际落点，包括前端详情标签、Excel 台账列头和正式规格文案。

## 成功标准（必填）

### 可度量结果

- **SC-001**：详情抽屉和 Excel 台账中不再出现“业务合规”这一用户可见术语。
- **SC-002**：相关前后端测试通过，并显式断言“业务风险分类”。
- **SC-003**：`001`、`002`、`004` 的当前正式规格不再互相冲突。
