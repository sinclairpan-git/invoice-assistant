# 发票整理助手手工验收清单

## 1. 启动服务

1. 后端启动：
   `UV_CACHE_DIR=/Users/sinclairpan/project/发票整理助手/.uv-cache uv run --project backend uvicorn backend.app.main:create_app --factory --host 127.0.0.1 --port 8000`
2. 前端启动：
   `COREPACK_HOME=/Users/sinclairpan/project/发票整理助手/frontend/.corepack corepack pnpm run dev -- --host 127.0.0.1 --port 5173`
3. 打开：
   `http://127.0.0.1:5173/`

## 2. 预置配置

在“首次配置向导”依次确认或新建以下生效版本：

- 税务档案：
  - `buyer_name = Shanghai Example Co`
  - `buyer_tax_no = 91310000X`
- 风险规则：
  - `minimum_confidence = 0.75`
  - `seller_whitelist = ["Acme Supplies Ltd", "Scan Services Co"]`
  - `review_keywords = ["DETAIL LIST ATTACHED"]`
- 命名模板：
  - `pattern = {date}_{amount}_{number}`

## 3. 上传样例

上传目录 `/Users/sinclairpan/project/发票整理助手/backend/tests/fixtures/invoices/` 下四张样例票：

1. `01-standard-electronic.pdf`
2. `02-scanned-ocr.pdf`
3. `03-review-required.pdf`
4. `04-duplicate.pdf`

## 4. 核心验收点

1. 工作台批次卡片显示：
   - 总文件数 `4`
   - 完成数 `4`
   - 失败数 `0`
   - 系统建议通过数 `2`
   - 合规票总金额 `384.50`
2. 结果页顶部同时显示：
   - 批次系统建议通过金额 `384.50`
   - 当前筛选系统建议通过金额
3. “系统建议通过”筛选下应看到两张票，重命名文件名分别为：
   - `20260417_128.50_STD-001.pdf`
   - `20260416_256.00_OCR-001.pdf`
4. “待复核”筛选下应看到 `03-review-required.pdf`，当前筛选系统建议通过金额应为 `0.00`
5. “疑似重复”筛选下应看到 `04-duplicate.pdf`，且不计入任何合规票金额汇总
6. 详情抽屉中应可查看：
   - 原始证据
   - 抽取字段
   - 购方校验记录
   - 风险标记
   - 疑似重复依据（仅重复票）
7. 对 `03-review-required.pdf` 发起人工复核“通过”后：
   - 留痕成功
   - `review_status` 更新为人工通过
   - `system_decision` 仍保持 `review_required`

## 5. 导出验收

依次执行三类导出并核对结果：

1. 系统建议通过 ZIP：
   - 文件数 `2`
   - 总金额 `384.50`
   - ZIP 内为两份重命名后的 PDF
2. 问题票 ZIP：
   - 文件数 `2`
   - 总金额 `216.50`
   - ZIP 内为 `03-review-required.pdf` 与 `04-duplicate.pdf`
3. Excel 台账：
   - 总记录数 `4`
   - 系统建议通过数 `2`
   - 系统建议通过总金额 `384.50`

## 6. 结论口径

- 合规票总金额只统计 `suggested_pass` 且排除疑似重复票。
- “批次总额”和“当前筛选总额”都必须展示。
- 人工复核只追加留痕，不改写系统判定。

## 7. Windows 便携版验收

### 7.1 基础启动

1. 将便携包放在带空格的目录，例如：
   `C:\Users\Public\Invoice Assistant Portable\`
2. 双击 `启动发票助手.bat`
3. 期望：
   - 可正常启动
   - 不因路径中有空格而失败
   - 浏览器自动打开首页；若未自动打开，优先访问 `http://127.0.0.1:18080`
   - 若 `18080` 被其他程序占用，则读取 `runtime/app.url` 中记录的实际地址，手动访问仍可进入首页

### 7.1A 自包含启动边界

1. 在干净 Windows 机器、非管理员账户下解压便携包
2. 不额外安装系统 Python 或 Node
3. 双击 `启动发票助手.bat`
4. 期望：
   - 可以直接启动
   - 不依赖系统现装 Python / Node
   - 非管理员账户也能正常写入 `data/`、`logs/`、`runtime/`

### 7.1B 中文路径

1. 将便携包放在中文目录，例如：
   `C:\Users\Public\发票助手便携版\`
2. 双击 `启动发票助手.bat`
3. 期望：
   - 可正常启动
   - 不因中文路径而失败

### 7.2 首次配置硬门槛

1. 在全新 `data/` 条件下启动
2. 不做配置，直接尝试上传发票
3. 期望：
   - 上传被阻止
   - 页面明确引导先完成“首次配置向导”
4. 完成首次配置后再次上传
5. 期望：
   - 上传恢复可用

### 7.2A 端口占用

1. 预先让其他程序占用 `127.0.0.1:18080`
2. 双击 `启动发票助手.bat`
3. 期望：
   - 服务不会失败退出
   - 会自动回退到 `18081-18089` 里的第一个可用端口
   - `runtime/app.url` 记录的是最终选择的实际地址
   - 浏览器自动打开或手动按 `runtime/app.url` 可进入首页

### 7.2B 浏览器未自动打开

1. 启动便携包
2. 若浏览器未自动打开，优先手动访问 `http://127.0.0.1:18080`
3. 若 `18080` 仍打不开，再查看 `runtime/app.url`
3. 期望：
   - 手动访问实际地址仍可进入首页
   - 用户指引中能找到这条备用操作

### 7.3 导出与打开导出文件夹

1. 完成一批发票处理
2. 分别执行建议通过 ZIP、问题票 ZIP、Excel 台账导出
3. 在“最近导出”区域点击“打开导出文件夹”
4. 期望：
   - 成功打开本地导出目录
   - 导出目录位于便携包 `data/exports/` 内
   - “最近导出”区域展示的是最近一次导出记录，而不是更早的旧路径

### 7.3A 扫描票 OCR

1. 上传扫描票样例 `02-scanned-ocr.pdf`
2. 等待处理完成
3. 期望：
   - OCR 路径可正常跑通
   - 扫描票能进入结果页并给出识别结论

### 7.4 停止脚本

1. 保持服务运行时双击 `停止发票助手.bat`
2. 期望：
   - 服务被停止
   - 再访问 `runtime/app.url` 最后记录的地址不可用
3. 仅关闭浏览器后不执行停止脚本，再重新双击启动脚本
4. 期望：
   - 能识别服务仍在运行或恢复访问
   - 不把“关闭浏览器”等同于“已停服务”

### 7.4A 陈旧 PID 恢复

1. 人为制造残留 `runtime/app.pid`
2. 确认对应进程已经不存在
3. 双击 `停止发票助手.bat`
4. 期望：
   - 提示“当前未检测到运行中的发票助手”
   - 清理陈旧 `app.pid` / `app.url`

### 7.5 `data/` 迁移复用

1. 在旧包中完成首次配置、上传、处理、导出
2. 停止服务
3. 仅复制旧包 `data/` 到新包
4. 启动新包
5. 期望：
   - 配置仍然存在
   - 历史批次仍可查看
   - 已导出的记录仍可见

### 7.5A 正式包升级

1. 使用上一正式便携版完成首次配置和一批处理
2. 停止服务并备份 `data/`
3. 解压下一正式便携版并恢复 `data/`
4. 启动新包
5. 期望：
   - 首页可用
   - 历史配置、批次、导出记录可继续读取
   - 若升级失败，可按维护文档回退到上一正式便携版

### 7.6 日常使用口径

1. 用户文档首页必须明确写出：
   - 第一次先完成首次配置向导
   - 关闭浏览器不等于停止服务
   - 平时用完双击 `停止发票助手.bat`
   - 浏览器没自动打开时先访问 `http://127.0.0.1:18080`，若失败再看 `runtime/app.url`
   - 第一次双击看到黑色窗口是正常现象

### 7.7 无口头解释独立演练

1. 只把压缩包和 `用户指引.html` 交给一位技术小白测试者
2. 不额外做口头解释
3. 观察其是否能独立完成：
   - 启动
   - 首次配置向导
   - 上传样例票
   - 导出并打开导出文件夹
   - 停止服务
   - 备份 `data/`
4. 期望：
   - 测试者能主要依靠 `用户指引.html` 走通流程
   - 不需要维护人员临场补充“第一次该去哪里”“怎么停服务”“导出文件在哪”这类关键指引
